## 输入

- `text_id`：文本的唯一整数 ID
- `content`：原始中文评论文本

## 你的任务

1. 只提取那些直接描述元宇宙/VR 游戏体验价值的短语。
2. 在保持原句顺序的前提下，用 `<p>...</p>` 包裹每个被提取的短语。
3. 为每个被提取的短语给出一个简洁、可复用的中文体验维度名称。
4. 让维度名称尽量贴近经验性表述，同时又能在不同评论之间复用。

## 维度命名参考

将以下示例作为风格锚点，而不是固定代码本：

- "画面有点暗" -> `设备画面`
- "戴头盔不舒服" -> `设备物理舒适度`
- "帮忙拍照指导" -> `增值服务` 或 `服务专业度`
- "挺有意思的" -> `游戏趣味性`
- "离家近" -> `位置`
- "活动价是30元" -> `价格`

## 维度规则

- 维度名称必须只使用中文。不要输出 `staff_patience` 或 `playfulness` 这样的英文标签。
- 每个短语对应一个主维度，写入 `normalized_label`。
- 尽量使用 2-6 个中文字符的简洁概念标签。
- 避免输出类似“`小哥哥很帅`”这类一次性标签，除非它能被抽象为可复用的体验维度。
- 如果一个短语只是笼统表扬而没有具体原因，只有在它明确体现体验价值时才编码，例如 `愉悦身心` 或 `推荐意愿`。

## 输出要求

只返回结构化输出：

- `content_with_labels`：原始内容，只在被提取短语外包裹 `<p>...</p>` 标签
- `items`：对象列表：
  - `name`：对提取短语的中文简短概括
  - `evidence`：原评论中的精确或近似原文证据
  - `normalized_label`：该条目的主中文维度
  - `reason`：一句简短中文说明，解释为何该证据支持该维度

## 质量检查清单

- 每个维度都必须有明确文本证据支撑。
- 每个维度名称都必须是中文。
- `content_with_labels` 只能使用 `<p>`；绝不能使用 `<e>`、HTML span、Markdown 或自定义标签。
- 保持输出简洁且稳定。
